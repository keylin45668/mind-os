# 任务迭代引擎

> 将用户任务的思考过程自动化迭代，加速收敛到高质量结论。
> 与迭代引擎（iterative-engine）的关系：迭代引擎 = 单轮人在场；本协议 = N 轮 AI 自动跑，人在检查点。
> 与 autoevolve 的关系：同构架构（文件驱动状态 + 标量指标 + 收敛条件），应用于用户任务而非系统合规。

---

## 触发方式

```yaml
trigger:
  命令: "/深度迭代 {任务描述}"
  自动: 迭代引擎单轮完成后，AI 判断问题尚未收敛时建议激活
  示例:
    - "/深度迭代 要不要从制造业转型做无人机表演"
    - "/深度迭代"（不带描述 → AskUserQuestion 询问任务）
```

---

## 三层架构

```
┌───────────────────────────────────────────────┐
│  Layer 1: 思考抽象（对话/输入 → 思考链文档）      │
│  提取：核心问题 + 论点链 + 关键假设 + 未解决问题   │
├───────────────────────────────────────────────┤
│  Layer 2: 自动迭代（文档 → N 轮快速模拟）         │
│  AI 蓝红双角色自动跑，每轮更新文档 + 质量信号      │
├───────────────────────────────────────────────┤
│  Layer 3: 人类检查点（迭代结果 → 人审核/注入）     │
│  人看摘要，注入新约束或确认收敛                    │
└───────────────────────────────────────────────┘
```

## Layer 1：思考抽象

### 输入来源

```yaml
input_sources:
  - 当前对话历史（用户描述 + 已有分析）
  - 用户直接给的描述（"/深度迭代 {任务描述}"）
  - 已有的 runtime/thinking/{task}/ 文件（如有）
```

### 抽象过程

AI 从输入中提取以下结构，生成 `thinking-trace.md`：

```markdown
# {任务名} — 思考链

## 核心问题（一句话）
{这件事的本质问题是什么}

## 关键假设
- [A1] {假设1}（状态: 未验证/已验证/已否决）
- [A2] {假设2}
- [A3] {假设3}

## 论点链
- [T1] {论点1} ← 依赖 [A1]
  - 支撑证据: {来源}
  - 红方攻击面: {潜在反驳方向}
- [T2] {论点2} ← 依赖 [A1, A2]
  - ...

## 关键转折点
- {对话中/分析中发生思路转变的地方}

## 未解决问题
- [U1] {待回答的问题}
- [U2] {缺少的信息}

## 当前结论（暂定）
{基于已有论点的初步结论}

## 质量基线
论点数: {N} | 假设验证率: {X}% | 未解决问题: {M}
```

### 抽象原则

```yaml
abstraction_rules:
  - 只提取推理链条，不保留对话形式（"你说...我说..."）
  - 每个论点必须标注依赖的假设（假设断了，论点就断了）
  - 本源检查：核心问题是否是"真问题"而非"表面问题"
  - 单文件 ≤ 1500 tokens（超限则压缩论点描述）
```

## Layer 2：自动迭代

### 迭代循环

```yaml
iteration_loop:
  max_rounds: 5              # 硬上限，防失控
  convergence_check: 每轮结束后

  per_round:
    step_1_blue:              # 蓝方：强化论点
      - 检查未解决问题 [U]，尝试回答
      - 检查未验证假设 [A]，尝试验证或找替代
      - 补充新论点（如有发现）

    step_2_red:               # 红方：攻击论点
      - 对每个存活论点找最强反驳
      - 首要攻击方向 = 本源检查（宪法第四条）
        "这个论点是在分析本质，还是在套框架？"
      - 攻击假设基础：假设 [A] 如果不成立，哪些论点倒下？
      - 提出新的未解决问题

    step_3_judge:             # 裁决
      - 论点存活/倒下/修正
      - 假设状态更新
      - 未解决问题更新

    step_4_update_trace:      # 更新思考链文档
      - 标记论点状态（SURVIVED/FALLEN/MODIFIED）
      - 更新假设验证状态
      - 更新质量信号

    step_5_convergence:       # 收敛检测
      - 计算质量信号
      - 判断是否收敛（见下方）
      - 收敛 → 退出循环
      - 未收敛 → 下一轮
```

> 质量信号、收敛条件、检查点格式见 `task-iterate-signals.md`（拆分以符合 ≤1000 tokens 预算）

## 与现有系统的集成

```yaml
integration:
  迭代引擎: 单轮仍可用；单轮未收敛时建议 /深度迭代
  输出模板: 最终输出遵循 output-template 格式
  会话评分: D2 应反映迭代轮数和收敛质量
  autoevolve: 同构但独立（系统规则 vs 用户任务）
```

## 文件位置

```yaml
files:
  协议: think/task-iterate.md（本文件）
  信号与收敛: think/task-iterate-signals.md
  过程文档: 对话中输出（默认），用户要求保存 → runtime/thinking/{task}/
  最终结论: 用户确认后 → {data}/content/ 或 knowledge/
```
