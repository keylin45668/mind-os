# 反脆弱方法论

> 来源：《反脆弱》《道德经》

---

## 三元分类器

```yaml
trigger: 新决策或新项目
protocol:
  1. 识别Top3可能冲击，估算每个冲击的收益/损失
  2. 判定：
     - 损失>收益 AND 不可逆 → 🔴脆弱
     - 损失≈收益 AND 有缓冲 → 🟡强韧
     - 收益>损失 AND 上限开放 → 🟢反脆弱
  3. 🔴 → 强制事前验尸 + Lollapalooza扫描
     🟡 → 正常流程，确保冗余
     🟢 → 快速通道，允许快速失败
组合健康度: 🔴 ≤ 20%，🟡 ≥ 40%，🟢 ≥ 20%
```

## 杠铃策略

```yaml
trigger: 资源分配决策
protocol:
  1. 强制两极化：安全端 + 冒险端
  2. 冒险端准入：最大损失已知 ≤ 投入，潜在收益 ≥ 5x
  3. 安全端不可被冒险端借用
  4. 比例定义在 data/identity/preferences.md 中
  5. 正业务增速 < 10% → 冒险端提升
     奇项目 IRR > 150% → 毕业进正池
     整体增速 > 30% → 正池提升至90%
```

## 选择权三要素

可放弃？损失封顶？收益开放？三要素全满足 → 有选择权 → 提升一个三元等级。

## 否定法

```yaml
negativa_rules:
  1. 减法优先：每次审计先问"可以删什么？"
  2. 新功能默认禁用
  3. 季度净效应评估
  4. 每季度清理：90天未引用的theory规则归档
  5. 知识库90天无链接的孤岛笔记归档
  6. 新增理论缺省拒绝，需满足：使命关联≥70% AND 未超预算
```

## 小压力源菜单

```yaml
扰动类型:
  🎲 随机挑战：用不常用模型重新分析近期决策
  🔄 逆向操作：对已确定决策生成最强反方论证
  📊 指标挑战：临时提高某指标基线10%一周
  🗑️ 减法实验：临时禁用一个theory模块一周
安全阈值: 损失<10%资源，可逆，每周≤1次
```

## 锁定逃逸

```yaml
trigger: 同一theory 6个月未变 AND 指标停滞 AND 用户不满
escape:
  a. 突变采样：从archive恢复已归档theory临时启用1周
  b. 回溯快照：回退到3个月前的theory快照
  c. 外部注入：引入最少使用学科的新模型
```
