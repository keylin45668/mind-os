## 使用场景与启动方式

```
场景 A：想（分析、决策、审查）
  启动目录: mind-os 所在目录（如 ~/git/）
  触发方式: 启动 Mind OS → 描述任务 → 自动匹配连接器
  AI 行为:  读取项目文件（只读）→ 用 theory 分析 → 对话中输出结论
  落盘:     默认不落盘；用户要求保存 → decisions/ 或 knowledge/

场景 B：做（修改制度文件、起草新模板）
  启动目录: 项目仓库目录（如 ~/git/zhidu-youhua/）
  触发方式: 项目自己的 CLAUDE.md / AI.md 生效
  AI 行为:  按项目规则直接修改文件
  落盘:     修改直接写入项目仓库

场景 C：边想边做（分析后直接改文件）
  启动目录: mind-os 所在目录
  触发方式: Mind OS 分析完 → 用户说"按这个结论帮我改"
  AI 行为:  通过 {repo} 路径直接修改项目文件
  落盘:     文件改动 → 项目仓库；决策记录 → Mind OS
```
