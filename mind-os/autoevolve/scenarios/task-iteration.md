# 场景集：任务迭代引擎（/深度迭代）

> 覆盖 think/task-iterate.md 三层架构的完整执行

---

## S-TITER-01：命令触发 + Layer 1 思考抽象

```yaml
user_input: "/深度迭代 要不要从制造业转型做无人机表演"
expected:
  - 命令路由识别: "/深度迭代" → think/task-iterate.md
  - 声明: "命令触发: /深度迭代 → task-iterate.md, MUST_RUN"
  - Layer 1 执行:
    - 提取核心问题（一句话，非用户原文复制）
    - 列出关键假设（≥ 2 条，标注状态）
    - 列出初始论点（≥ 3 条，每条标注依赖的假设）
    - 列出未解决问题（≥ 1 条）
    - 本源检查: 核心问题是否是"真问题"
  - 产出 thinking-trace 结构（不要求落盘，但结构必须完整）
checks: [F01, F02, F03, F04]
```

## S-TITER-02：Layer 2 自动迭代循环

```yaml
user_input: "/深度迭代 我的团队应该采用 OKR 还是 KPI 考核体系"
expected:
  - Layer 1 完成后进入 Layer 2
  - 每轮包含:
    - step_1_blue: 蓝方强化/补充论点（可见蓝方新论点或证据）
    - step_2_red: 红方攻击（首要攻击 = 本源检查，与蓝方观点不同）
    - step_3_judge: 裁决（明确标注 SURVIVED/FALLEN/MODIFIED）
    - step_4_update: 质量信号更新（论点存活率、新发现数）
  - 至少跑 2 轮（1 轮不够证明"迭代"）
  - 红方不能与蓝方意见一致（深度检查: 红方必须提出不同观点）
  - 收敛检测: 明确声明收敛原因或达到上限原因
checks: [F05, F06, F07, F08, F09]
```

## S-TITER-03：Layer 2 质量信号计算

```yaml
user_input: "/深度迭代 是否应该把产品价格降低 30% 抢市场份额"
expected:
  - 每轮结束后展示质量信号:
    - 论点存活率: 有具体百分比（非"大部分存活"）
    - 新发现数: 有具体数字
    - 假设验证率: 有具体百分比
    - 本源偏离度: 有 1-5 评分
  - 信号数值变化可追踪（R1 → R2 有变化）
  - 收敛条件检查: 明确引用收敛标准（连续 2 轮新发现=0 等）
checks: [F07, F08]
```

## S-TITER-04：Layer 2 异常退出 — 论证崩塌

```yaml
user_input: "/深度迭代 用占星术来选择投资标的"
expected:
  - Layer 1: 核心问题提取（即使问题本身有问题也先提取）
  - Layer 2 R1: 红方应在首轮本源检查时严重质疑前提
  - 论点存活率应 < 30%
  - 触发强制停止: "论证基础崩塌，建议重新定义问题"
  - 不继续盲目迭代（不能假装论证成立跑完 5 轮）
checks: [F09, F10, E04]
```

## S-TITER-05：Layer 3 人类检查点

```yaml
user_input: "/深度迭代 团队是否应该从线下转为远程办公"
expected:
  - 迭代完成后展示结构化结果:
    - 一句话结论
    - 存活论点列表（带轮次标注）
    - 被推翻论点列表（带原因）
    - 关键风险
    - 未解决问题
    - 迭代轨迹摘要（每轮一行）
  - 调用 AskUserQuestion 提供 4 选项:
    - "确认结论" / "注入新约束再跑" / "展开详细" / "重新定义问题"
  - 不直接结束，等用户回应
checks: [F11, F12]
```

## S-TITER-06：Layer 3 注入新约束后重新迭代

```yaml
context: S-TITER-05 完成后，用户选"注入新约束"
user_input: "加一个约束: 远程办公必须保证核心时间段全员在线"
expected:
  - 约束被写入 thinking-trace 的假设区
  - 所有现有论点被用新约束重新审计（哪些受影响）
  - 重新进入 Layer 2 迭代（轮次计数可重置或续接，需声明）
  - 新一轮迭代结果反映约束的影响
checks: [F13]
```

## S-TITER-08：与迭代引擎的衔接

```yaml

user_input: "帮我分析线上线下混合销售模式的利弊"
expected:

  - 如果 AI 判断问题未充分收敛（多方向未全部评估、红方有强未回应质疑）

  - 不强制，用户选择后才执行
checks: [F14]
```

## S-TITER-09：max_rounds 硬上限

```yaml
user_input: "/深度迭代 如何在 5 年内将公司从 500 万做到 5 亿"
expected:

  - 不无限跑下去
checks: [F09]
```
