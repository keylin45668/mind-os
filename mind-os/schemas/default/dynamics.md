## 2. 存量-流量模型

```
流入 ──→ [知识库K] ──→ 流出
K变化率 = 流入 - 流出
```

当流入 > 流出，K 增长；增强回路下指数增长；调节回路防止无限膨胀。

## 4. 系统陷阱免疫（架构级）

| 陷阱模式 | 本质 | 免疫措施 |
|---------|------|---------|
| 目标侵蚀 | 标准被实际绩效拖低 | metrics.md 版本化，修改须记录原因 |
| 转嫁负担 | 短期修复削弱长期能力 | 人类参与度 ≥ 50% 硬约束 |
| 信息过载 | 流入 > 处理能力 | inbox 断路器：积压 > 阈值自动降级 |
| 共演化锁定 | 人-AI 过度耦合 | 每周探索性噪声 + 季度收敛检测 |

具体陷阱识别规则（含扩展陷阱）定义在当前 theory pack 中。

## 5. 混沌边缘检测

```yaml
order_signal:   # 太有序——僵化
  - AI输出相似度 > 阈值
  - 当前 theory pack 长期零修改
  action: 增加随机性（触发小压力源）

chaos_signal:   # 太混乱——失控
  - AI输出连续被否决
  - 并行进程过多
  action: 加强约束（回读 constitution，启用最小介入模式）

healthy: 两个信号均未触发
```
