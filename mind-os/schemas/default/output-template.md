# 输出文档模板（MUST_RUN）

> 所有迭代引擎的最终输出必须遵循本模板。
> 核心原则：**总纲言简意赅，详情分章节单独文件，单文件 ≤ 1500 tokens。**

---

## 文件结构

```
runtime/thinking/{task}/output/
  ├── summary.md              ← 总纲（≤ 800 tokens，一页纸能读完）
  ├── direction-d1.md         ← 方向1 详细分析
  ├── direction-d2.md         ← 方向2 详细分析
  ├── direction-d3.md         ← 方向3 详细分析
  ├── adversarial-d1.md       ← 方向1 红蓝对抗记录
  ├── adversarial-d2.md       ← 方向2 红蓝对抗记录
  ├── comparison.md           ← 五维对比表
  └── appendix.md             ← 附录（数据来源、计算过程，可选）
```

对话模式下不创建文件，但输出内容仍然按总纲→章节结构组织。

---

## 总纲模板（summary.md）

```markdown
# {任务名} — 分析总纲

> 日期: {date} | 方向: {N}个 | 轮次: {M}轮 | 信心度: {X}%
> 收敛原因: {red_exhausted / all_resolved / ...}

## 一句话结论
{用一句话回答用户的核心问题，不超过 50 字}

## 推荐方案
{推荐方向} — {一句话理由}
- 存活率: {X}% | 信心度: {Y}/5
- 核心优势: {1-2 点}
- 核心风险: {1-2 点}

## 各方向速览
| 方向 | 结论 | 存活率 | 状态 |
|------|------|--------|------|
| d1: {名称} | {一句话} | {X}% | {converged/excluded} |
| d2: {名称} | {一句话} | {X}% | {converged/excluded} |

## 下一步行动
1. {具体行动项 + 截止时间}
2. {具体行动项 + 截止时间}

## 详细章节
- [方向1 详细分析](direction-d1.md)
- [方向2 详细分析](direction-d2.md)
- [五维对比表](comparison.md)
- [红蓝对抗记录](adversarial-d1.md)

---
分析溯源: 使用了 {theory 文件列表}，匹配模式 {模式×拓扑}
```

**硬约束：总纲 ≤ 800 tokens。超过必须拆到详情章节。**

## 方向详情模板（direction-{id}.md）

```markdown
# 方向 {id}: {方向名称}

> 状态: {converged/excluded} | 轮次: {N} | 信心度: {X}%
> 收敛原因: {原因}

## 本源
{这件事的本质是什么}

## 最终论点（经红方验证）
- [SURVIVED:B-xxx] {论点内容} ✅
- [SURVIVED:B-xxx] {论点内容} ✅
- [MODIFIED:B-xxx] {论点内容}（修正: {原因}）⚠️

## 被否决的论点
- [FALLEN:B-xxx] {论点内容}（原因: {红方攻破理由}）❌

## 关键风险
1. {风险描述} — 来源: [R-xxx]
2. {风险描述} — 来源: [R-xxx]

## 成功条件
1. {条件}
2. {条件}
```

**硬约束：≤ 1500 tokens/文件。**

## 红蓝对抗记录模板（adversarial-{id}.md）

```markdown
# 方向 {id} — 红蓝对抗记录

## 逐轮摘要
| 轮次 | 蓝方论点 | 红方新质疑 | 存活/倒下/修正 | 信心度 |

| R1   | 3       | 3 [NEW]  | 1/1/1        | 33%    |
| R2   | 2+回应3 | 0 [NEW]  | 3/0/0        | 100%   |

## 关键转折点
- R1→R2: 蓝方补充了 {关键证据}，红方无新弹药 → 收敛

## 全部编号索引
（供溯源查证，每条一行）
[B-d1r1-01] {论点摘要}
[R-d1r1-01] [NEW] [严重] {质疑摘要}
[RESP:R-d1r1-01] {回应摘要}

```

## 强制执行规则

```yaml
output_enforcement:
  # 总纲
  summary_max_tokens: 800
  summary_required_sections:
    - "一句话结论"
    - "推荐方案"
    - "各方向速览"
    - "下一步行动"
    - "详细章节"       # 必须有链接/引用到分章节

  # 分章节
  chapter_max_tokens: 1500
  chapter_types:
    - direction: 每个方向一个文件
    - adversarial: 每个方向的对抗记录一个文件
    - comparison: 五维对比一个文件
    - appendix: 可选

  # 检查（由评分 Agent 或 output-validator hook 执行）
  validation:
    - summary.md 存在且有全部 5 个必选章节？
    - summary.md ≤ 800 tokens？
    - 每个 active 方向都有 direction-{id}.md？
    - direction-{id}.md 中有 [SURVIVED:] 编号引用？
    - comparison.md 有五维评分表？
    - 所有章节文件 ≤ 1500 tokens？
    违规: 输出不完整，要求补全后再提交

  # 对话模式适配
  conversation_mode:
    - 先输出总纲内容（相当于 summary.md 的内容）
    - 总纲末尾说: "以下是各方向详细分析:"
    - 然后按 direction → comparison 顺序输出
    - 每个章节用 "---" 分隔
    - 如果用户要求保存 → 按上述文件结构落盘
```
