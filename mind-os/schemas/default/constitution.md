# 宪法（三条，不可删除，只可追加）

**第一条**：一切认知活动 = `Transform(input, context) → output`，其中 `context = schema ⊕ theory ⊕ data`，三者正交，独立可替换。

**第二条**：系统自身必须可迭代，但成功的定义不可被 AI 篡改。`Iterate(system) = 审计 → 迭代 → 验证 → 审计 → …` 评估指标由人类冻结，AI 只能在冻结指标下优化。

**第三条**：系统宁简勿繁——新增任何规则前必须先删除或归档一条旧规则。复杂度预算：schema ≤ 30 条核心规则，theory ≤ 15 个活跃模块。超限即冻结新增。

---

## 涌现原则

三条宪法 + 三层分离就是"少量简单规则"。具体协议是涌现的参考实现而非硬约束——运行时只需加载 schema/ 核心文件，复杂行为由人-AI 交互自然涌现。

---

## 元操作定义

所有认知活动同构为一个元操作 `Transform` 的参数预设：

| 活动 | input | context.rules | output |
|------|-------|---------------|--------|
| 收集 | 原始信息 | 收集规则 | inbox 条目 |
| 分类 | inbox 条目 | 分类规则 | 归档位置 |
| 决策 | 选项集 | 决策原则 | 选定方案 |
| 回顾 | 历史记录 | 回顾模板 | 模式洞察 |
| 检索 | 查询意图 | 知识库索引 | 相关内容 |
| 存储 | 处理后内容 | 存储规则 | 持久化记录 |

## 三层分离

```
context = schema ⊕ theory ⊕ data
```

| 维度 | 是什么 | 类比 | 替换操作 |
|------|--------|------|---------|
| schema | 管道怎么连、阶段有哪些 | OS内核 | 改工作流结构 |
| theory | 用什么方法处理 | 应用算法 | 改处理规则 |
| data | 谁在用、处理什么 | 用户文件 | 改处理对象 |

正交性承诺：换 theory → schema/data 不变；换 data → schema/theory 不变；换 schema → theory/data 不变。
