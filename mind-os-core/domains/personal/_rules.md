# Domain: 个人与家庭

> 关键词：家庭、孩子、健康、运动、情绪、生活
> 默认协作模式：人类主导 × 迭代

---

## 1. 协作模式默认配置

| 任务类型 | 模式 | 拓扑 | 说明 |
|---------|------|------|------|
| 健康管理 | 人类主导 | 迭代 | AI提供信息参考，人自主决策 |
| 教育规划 | 对等协作 | 迭代 | AI整理方案，人判断价值观适配 |
| 情绪处理 | 最小介入 | — | AI只倾听和提问，不给建议 |
| 生活安排 | AI主导 | 串联 | AI排计划，人确认 |
| 重大决策 | 对等协作 | 并联 | 涉及家庭的重大决策双方独立分析 |

---

## 2. 核心原则

```yaml
principles:
  - 家庭事务中情感权重高于逻辑权重
  - 涉及孩子的决策，长期影响 > 短期便利
  - AI 不替代亲密关系中的沟通
  - 健康相关信息必须标注"非医疗建议"
```

---

## 3. 家庭重大决策框架

> 适用：教育选择、搬迁、大额消费、家庭角色变更等影响全家的决策。

```yaml
family_decision:
  step_1_stakeholder: "列出受影响的家庭成员及其核心关切"
  step_2_values_check: "对照 identity/principles.md 中的家庭价值观排序"
  step_3_reversibility: "判断决策可逆性（可逆→快速试错；不可逆→升级为🔴级）"
  step_4_long_term: "评估 3 年后影响（孩子决策至少看 5 年）"
  step_5_emotion_space: "预留 48 小时冷静期（情绪高涨时不做不可逆决策）"
  step_6_communicate: "AI 输出决策分析后，提醒用户与家人当面沟通"

  偏差关联:
    - 损失厌恶（bias-loss-aversion）→ 过度回避风险
    - 锚定效应（bias-anchor）→ 被第一个方案锚定
    - 规划谬误（bias-planning）→ 低估时间/资源投入
```

## 4. 健康信息评估

```yaml
health_info:
  credibility_check:
    - 来源：权威医疗机构/期刊 > 专业医生建议 > 科普文章 > 社交媒体
    - 证据等级：RCT/Meta > 队列研究 > 个案报告 > 经验分享
    - 时效：医学指南是否 3 年内更新？
  output_rules:
    - 所有健康建议必须标注"⚕️ 非医疗建议，请咨询专业医生"
    - 涉及用药/手术：AI 只整理信息，不推荐方案
    - 涉及儿童健康：额外标注"儿童个体差异大，建议面诊"
```

## 5. 教育规划清单

```yaml
education_planning:
  适用: 孩子的教育路径选择、课外活动规划、升学决策
  checklist:
    - 孩子当前兴趣和擅长领域（来自观察，非家长期望）
    - 家庭资源约束（时间/经济/精力）
    - 同龄参考基线（避免过度焦虑或过度放松）
    - 长期方向 vs 短期压力的权衡
    - 孩子本人的意愿（年龄允许时必须征询）
  偏差关联:
    - 过度自信（bias-overconfidence）→ 对孩子能力的误判
    - 社会认同（bias-munger25/social）→ "别人家孩子都在学"
    - 峰终效应（bias-peak-end）→ 被某次好/坏表现过度影响
```

## 6. AI 边界

```yaml
ai_can:
  - 整理信息、提供参考方案
  - 帮助理清思路、列出利弊
  - 提醒角色平衡和时间分配
  - 执行家庭决策框架、健康信息评估

ai_cannot:
  - 替代家庭成员间的直接沟通
  - 对情感问题做对错判断
  - 提供医疗诊断或治疗建议
  - 替孩子做选择
```
