# Domain: 投资与财务

> 关键词：投资、融资、财务、预算、估值、现金流、ROI
> 默认协作模式：对等协作 × 并联

---

## 1. 协作模式默认配置

| 任务类型 | 模式 | 拓扑 | 说明 |
|---------|------|------|------|
| 投资决策 | 对等协作 | 并联 | 双方独立分析后合并，防单一视角 |
| 预算编制 | 人类主导 | 迭代 | 人定框架，AI填充数据和测算 |
| 财务分析 | AI主导 | 串联 | AI先整理数据，人判断含义 |
| 融资谈判 | 人类主导 | 迭代 | AI提供估值参考，人做最终判断 |
| 现金流管理 | AI主导 | 迭代 | AI监控异常，人决定动作 |

---

## 2. 投资决策必检清单

```yaml
before_invest:
  1. 三元分类: "损失封顶？收益开放？可逆？"
  2. 能力圈检查: "这个领域你有认知优势吗？"
  3. WYSIATI检查: "信息来源有几个？完备度？"
  4. 事前验尸: "假设失败了，最可能的原因？"
  5. 杠铃检查: "这笔投资占总资产比例？符合杠铃策略吗？"
```

## 3. 财务专属偏差检查

```yaml
anchoring_on_price:
  trigger: 讨论估值时过度锚定报价
  protocol: "忽略报价，从基本面独立估值"

sunk_cost_trap:
  trigger: "已经投了X万"作为追加理由
  protocol: "归零测试——如果从头来，还会投吗？"

overconfidence_in_returns:
  trigger: 收益预期缺少区间
  protocol: "给出最好/最可能/最差三个场景"
```
