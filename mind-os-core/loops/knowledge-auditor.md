# Layer 3: 知识审计员

> 频率：会话结束时或用户触发 | 阻断级别：软提示 | 作用域：`runtime/`、`data/knowledge/`、项目连接器

## 目标

在任务收尾阶段检查运行时状态、知识引用和项目连接是否老化，避免“当前回答正确，但底层上下文已过期”。

## 检查项

1. **runtime 状态文件**
   - `runtime/` 内状态文件是否长期未更新、与当前会话不一致或已失效
   - 重点看：焦点、面板、审计 backlog、临时状态
2. **knowledge 事实有效性**
   - `data/knowledge/` 中本次引用的事实、结论、外部说明是否仍成立
   - 对明显依赖时间、版本、组织结构的条目做过期标记
3. **连接器可访问性**
   - 项目连接器里的 `context_files` 是否仍存在、路径是否可读
   - 若连接器指向的 repo / 文件已移动，标记失效

## 响应协议

- 不阻断当前开发或回答
- 生成“过期清单”并提示用户确认
- 建议动作只给到“确认 / 更新 / 删除 / 忽略本次”四类，不替用户擅自改知识库

## 输出模板

```text
Knowledge Auditor
- runtime 过期项: {无 / 列表}
- knowledge 待确认事实: {无 / 列表}
- connector 失效项: {无 / 列表}
- 建议: 请用户确认后再清理或更新
```

原则：低频审计以暴露老化为主，不在未经确认时自动改写知识资产。
