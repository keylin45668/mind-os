# Layer 2: 协议守卫

> 频率：每 3-5 轮交互 | 阻断级别：分级 | 作用域：当前任务执行链

## 目标

审计本轮会话是否真的按 Mind OS 协议执行，而不是只口头声明“已分析”“已按流程处理”。

## 核心检查

1. **MUST_RUN 完整性**
   - 命中的 theory / 命令是否真的执行了 `MUST_RUN` 协议？
   - 若跳过 `SHOULD`，是否说明理由？
2. **Pre-Output Gate**
   - 是否声明了匹配文件、协作模式、拓扑？
   - 是否做了“我是否按协议执行”的输出前自检？
3. **分级一致性**
   - 开始说的是轻量回答、标准分析还是深度迭代？
   - 实际输出深度是否与一开始声明一致，还是中途偷偷降级或空转？

## 响应协议

- `⛔ 暂停执行，回溯修正`
  - 条件：`MUST_RUN` 漏执行、门控被跳过、深度等级严重失真
  - 动作：停止继续回答，回溯到最近一次协议断点，补执行后再继续
- `⚠️ 记录偏差`
  - 条件：协议基本成立，但存在轻微降级、声明不清、理由不足
  - 动作：记录偏差点，并在后续回答中修正

## 审计输出模板

```text
Protocol Guardian
- 状态: ✅ / ⚠️ / ⛔
- 偏差点: {无 / 列表}
- 修正动作: {继续 / 追加声明 / 回溯补执行}
```

原则：协议真执行比“看起来很完整”更重要。
