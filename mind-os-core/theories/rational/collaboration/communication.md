# 沟通策略方法论

> 来源：金字塔原理 / BATNA-ZOPA / 非暴力沟通 / STAR 法
> 核心原则：**先理解对方的立场，再设计你的表达**

---

## 触发

```yaml
trigger: 说服/谈判/汇报/沟通/表达/演讲/怎么说/怎么谈/邮件策略
```

## 协议

### Step 1：受众分析（必做）

```yaml
audience:
  1. 对方是谁: 角色/决策权/知识背景
  2. 对方关心什么: 核心利益/担忧/KPI
  3. 对方的现有立场: 支持/中立/反对/未知
  4. 信息不对称: 你知道但对方不知道的 / 对方知道但你不知道的
```

### Step 2：目标定义

```yaml
goal:
  底线目标: 最低可接受结果（BATNA — 谈崩了的替代方案）
  理想目标: 最佳结果
  隐性目标: 关系维护/信任建立/长期合作
  判断: 本次沟通是一次博弈还是重复博弈？→ 重复博弈中让步是投资
```

### Step 3：策略选择

```yaml
strategy:
  汇报/提案（对上）:
    结构: 结论先行 → 支撑论据 → 风险预案 → 请求决策
    原则: 领导要的是判断不是过程；先说答案再解释

  说服/推动（对等）:
    结构: 共同利益 → 问题定义 → 方案 → 对方获益 → 行动号召
    原则: 从对方的关切出发，不是从你的需求出发

  谈判（利益冲突）:
    结构: 建立共识区 → 识别分歧 → 交换（你的低成本=对方高价值）→ 锚定
    原则: 先扩大蛋糕再分蛋糕；第一个出价的人设定锚点

  高难度对话（冲突/坏消息）:
    结构: 事实描述（非评判）→ 影响说明 → 感受表达 → 请求
    原则: 对事不对人；"我观察到X"而非"你总是X"
```

### Step 4：预演

```yaml
rehearsal:
  1. 模拟对方可能的反应（至少 2 种）
  2. 准备回应（特别是反对意见）
  3. 识别地雷: 哪些话题/措辞会触发对方防御？→ 绕开或重新框架
```

## 输出格式

```
🎯 沟通目标: {一句话}
👤 受众: {角色} — 关心 {核心利益}
📋 策略: {汇报/说服/谈判/高难度对话}

话术框架:
  开场: {怎么开头}
  核心论点: {1-3 个，按对方关切排序}
  预判反对: {对方可能说什么 → 你怎么回}
  收尾: {明确的行动号召或下一步}
```

## 防退化

- 没做受众分析就给话术 → 无效，先补 Step 1
- 从"我要什么"而非"对方关心什么"出发 → 违反本源（宪法第四条）
