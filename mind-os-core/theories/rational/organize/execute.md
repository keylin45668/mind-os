## 摘要

- **目标分解**：O(目标)→KR(可量化结果)→任务→最小可行步骤
- **关键路径**：识别依赖链→找最长路径→优先攻关
- **第一步锚定**：每个计划必须定义"现在就能做的第一步"，消除启动阻力
- **PDCA循环**：Plan→Do→Check→Act，每轮迭代更新任务状态

# 执行落地协议

---

## 触发

```yaml
trigger: 执行/计划/拆解/步骤/排期/落地/OKR/怎么做到/行动方案
input: 已确定的决策或目标（通常来自 iterative-engine / antifragile 的输出）
```

## 协议

### Step 1：目标澄清（30 秒）

```yaml
clarify:
  1. 成功画面: "做到什么程度算成功？" → 可验证的终态描述
  2. 时间框架: 截止时间或里程碑节奏
  3. 资源盘点: 可用的人/钱/时间/工具
  4. 约束: 不可动的条件
```

### Step 2：里程碑分解

```yaml
decompose:
  1. 从终态倒推 → 拆成 3-5 个里程碑（每个有明确交付物）
  2. 每个里程碑再拆成具体任务（每个任务 ≤ 2 小时可完成）
  3. 标注依赖关系: 哪些必须串行？哪些可并行？
  4. 识别关键路径: 最长串行链 = 最短完成时间
```

### Step 3：第一步锚定

```yaml
first_step:
  问: "今天能做的最小一步是什么？"
  要求:
    - 具体到可以立刻动手（不是"调研一下"，而是"打开X网站搜索Y"）
    - 耗时 ≤ 30 分钟
    - 完成后有可见产出
  原理: 启动成本是最大阻力，最小步骤消除启动摩擦
```

### Step 4：检查点设计

```yaml
checkpoints:
  1. 每个里程碑设一个检查点
  2. 检查内容:
     - 进度: 按计划/超前/滞后？
     - 假设: 当初的假设还成立吗？
     - 调整: 需要改路线还是继续？
  3. 检查点不通过 → 触发 decide/rules.md 重新排优先级
```

## 输出格式

```
🎯 目标: {一句话}
📅 截止: {日期}

里程碑:
  M1: {交付物} — {日期}
    □ 任务1（依赖: 无）
    □ 任务2（依赖: 任务1）
  M2: {交付物} — {日期}
    □ 任务3（依赖: M1）
    □ 任务4（依赖: 无，可并行）

⚡ 今天第一步: {具体行动}
📋 下次检查: {日期} — 检查 M1 是否达成
```

## 防退化

- 里程碑 > 5 个 → 过度拆分，合并
- 任何任务 > 2 小时 → 继续拆
- 没有"今天第一步" → 未完成，补上
