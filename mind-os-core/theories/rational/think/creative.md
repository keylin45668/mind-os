# 创造力引擎

> 来源：SCAMPER / 类比迁移 / 约束创造力 / 发散-收敛模型
> 核心原则：**先发散不评判，再用决策模块收敛筛选**

---

## 触发

```yaml
trigger: 设计/创意/方案/产品/点子/命名/想个/怎么做
protocol: 先发散（本模块）→ 再收敛（iterative-engine 或 antifragile）
```

## 发散阶段（不评判，只展开）

### 1. 约束提取

```yaml
constraints:
  1. 明确不可变约束（预算/时间/技术/法规）
  2. 明确可变约束（假设/惯例/偏好）
  3. 追问: "这个约束真的不可变吗？" → 可变约束是创意入口
```

### 2. 四种发散路径（至少用 2 种）

```yaml
paths:
  类比迁移:
    "其他领域怎么解决类似问题？"
    从至少 2 个不相关领域借鉴 → 映射到当前问题
    示例: 餐饮排队→迪士尼排队体验设计→产品等待页优化

  逆向设计:
    "如果要让这件事彻底失败，怎么做？"
    列出失败路径 → 逐条反转 → 得到成功路径

  SCAMPER:
    S-替换/C-组合/A-调整/M-放大缩小/P-改用途/E-消除/R-重排
    对现有方案逐条过一遍，每条至少产出 1 个变体

  极端约束:
    "如果只有 1/10 的预算怎么做？"
    "如果必须明天上线怎么做？"
    极端约束迫使跳出常规路径
```

### 3. 产出要求

- 至少 **5 个差异化方案**（不是同一思路的微调）
- 每个方案一句话描述 + 核心假设

## 收敛阶段

```yaml
converge:
  1. 用宪法第四条检查: 每个方案的本源是什么？解决的是真问题还是假问题？
  2. 快速筛: 不可变约束淘汰 → 剩余方案
  3. 深度评: 剩余方案进入 iterative-engine（红蓝对抗）或 antifragile（三元分类）
  4. 输出: 推荐方案 + 备选 + 被淘汰方案及原因
```

## 防退化

- 发散阶段禁止说"不可行"——可行性判断属于收敛阶段
- 类比迁移必须跨领域（同领域参考不算类比）
- 方案数 < 5 → 未完成发散，继续
