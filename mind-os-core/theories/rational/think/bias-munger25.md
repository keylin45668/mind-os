## 场景触发映射

```yaml
# ── 已有场景（保留）──

social_influence:
  trigger: 团队决策/集体讨论/投票
  芒格条目: "#14 社会认同 + #25 权威服从"
  checks: [社会认同偏差, 权威服从偏差]
  protocol: 匿名独立判断 → 再公开讨论
  示例: "大家都觉得应该投资这个项目" → 检查是否因为多数人支持而放松审查

incentive_bias:
  trigger: 设计薪酬/绩效/激励/合同条款
  芒格条目: "#1 激励机制偏差"
  checks: [激励是否与期望行为对齐, 是否存在反向激励]
  protocol: |
    1. 列出激励结构 → 2. 推演"如果我是被激励方，会做什么" → 3. 检查是否激励了不期望的行为
  示例: "销售提成按签约金额算" → 可能激励签大单但不管回款

consistency_bias:
  trigger: 评估"是否坚持原路线"/沉没成本/战略转向
  芒格条目: "#4 避免不一致倾向"
  checks: [一致性偏差, 承诺升级]
  protocol: 归零测试 + 反转思维
  示例: "我们已经投了200万，不能放弃" → 归零：如果今天从零开始，还会投吗？

# ── 新增场景 ──

liking_disliking:
  trigger: 评价某人/评估合作方/人事决策/合作伙伴选择
  芒格条目: "#2 喜爱/讨厌倾向"
  checks: [因喜欢某人而高估其方案, 因讨厌某人而低估其建议]
  protocol: |
    1. 标注"我对这个人的好感/恶感程度（1-5）"
    2. 好感≥4 或 恶感≥4 → 触发：将方案匿名化后重新评估
    3. 匿名评估结果 vs 原始判断差异 > 1 级 → 标记"⚠️ 情感干扰"
  示例: "张总人挺好的，他的方案应该靠谱" → 方案质量和人品无关

reciprocity:
  trigger: 对方给了好处后要求决策/谈判中的让步/免费试用后购买
  芒格条目: "#5 回馈倾向"
  checks: [是否因收到好处而做出不对等的让步]
  protocol: |
    1. 识别"对方给了我什么"
    2. 将回馈情感搁置，独立评估"这个决策本身值不值得"
    3. 如果去掉回馈因素后结论翻转 → 标记"⚠️ 回馈偏差"
  示例: "他请我吃了几次饭，我不好意思拒绝合作" → 饭局成本 vs 合作风险不对等

doubt_avoidance:
  trigger: 时间紧迫下的快速决策/压力下的承诺/被催促做决定
  芒格条目: "#3 避免怀疑倾向"
  checks: [是否因为不想承受不确定性而过早下结论]
  protocol: |
    1. 检查"是否有人在催你做决定"
    2. 问："如果推迟 24 小时决定，最坏结果是什么？"
    3. 最坏结果可接受 → 推迟；不可接受 → 执行，但记录"⚠️ 压力决策"
  示例: "今天必须签字" → 真的必须今天吗？deadline 是真是假？

lollapalooza:
  trigger: 重大决策中多个心理偏差同时出现/情绪异常高涨/异常一致的集体判断
  芒格条目: "#24 多因素叠加效应（Lollapalooza）"
  checks: [是否有 ≥3 个偏差同时作用于同一决策]
  protocol: |
    1. 列出当前决策涉及的所有偏差（从本次会话的偏差检查结果中提取）
    2. 偏差数 ≥ 3 且方向一致（全部推向同一结论）→ 触发 Lollapalooza 警报
    3. 警报后：强制执行 /反脆弱 + /跨审（独立复审）
  示例: 社会认同(大家都说好) + 回馈(对方请客) + 激励(成交有提成) → 三力合一推向"签约"

output: "🧲 芒格检查：场景={scenario}，命中偏差={biases}，对策={actions}"
```
