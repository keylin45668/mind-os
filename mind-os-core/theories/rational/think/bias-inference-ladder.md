## 摘要

- **6层模型**：从行动(顶)→信念→结论→解释/归因→选择性数据→原始事实(底)逐层拆解
- **标注**：每层 ✅有依据/⚠️缺证据/❌纯假设，薄弱环节≥2个→推论链不可靠
- **双环学习**：红环(回到事实层补数据) + 蓝环(回到解释层生成替代解释) → 交汇更新结论
- **触发**：人际冲突时升级为 MUST_RUN

# 推论阶梯

---

## 触发条件

```yaml
trigger:
  关键词: 我觉得/我认为/他肯定是/他们总是/不理解/为什么他要
  场景: 人际冲突 OR 团队矛盾 OR 归因推测 OR 对他人动机的判断
执行级别: SHOULD（人际冲突升级为 MUST_RUN）
```

---

## 阶梯模型（从底到顶）

```
  ┌─────────────────────┐
  │  6. 行动             │ ← 基于信念采取的行动
  │  5. 信念             │ ← 对世界的结论性判断
  │  4. 结论             │ ← 从解释中得出的推断
  │  3. 解释/归因         │ ← 对选择的数据赋予含义
  │  2. 选择性数据        │ ← 从观察中选择了什么
  │  1. 原始观察/事实      │ ← 可验证的客观事实
  └─────────────────────┘
```

---

## 6 步拆解协议

当检测到推论阶梯触发条件时，执行以下步骤：

```yaml
step_1_identify_action:
  问: "你准备做什么？或已经做了什么？"
  输出: 具体行动描述
  标注: —（行动层，起点）

step_2_surface_belief:
  问: "你为什么要这么做？背后的信念/判断是什么？"
  输出: "我认为…" 格式的信念陈述
  标注: ✅ 有依据 / ⚠️ 缺证据 / ❌ 纯假设

step_3_trace_conclusion:
  问: "这个信念是怎么得出的？你推断了什么？"
  输出: 从现象到结论的推理链
  标注: ✅ 逻辑严密 / ⚠️ 有跳跃 / ❌ 因果倒置

step_4_examine_interpretation:
  问: "你是怎么解读这些现象的？有没有其他解读方式？"
  输出: 至少 2 种不同解释（原始解释 + 替代解释）
  标注: ✅ 多元解释 / ⚠️ 只有一种解释 / ❌ 解释=事实混淆

step_5_check_data_selection:
  问: "你注意到了什么？忽略了什么？"
  输出: 列出已选择的数据 + 可能被忽略的数据
  标注: ✅ 数据全面 / ⚠️ 有选择性偏差 / ❌ 只看到想看的

step_6_ground_in_facts:
  问: "最底层的可验证事实是什么？"
  输出: 纯事实陈述（可被第三方验证）
  标注: ✅ 可验证 / ⚠️ 部分可验证 / ❌ 无法验证
```

---

## 诊断规则

```yaml
diagnosis:
  薄弱环节计数: 统计 ⚠️ 和 ❌ 的数量
  rule_1: "⚠️ 或 ❌ ≥ 2 → 推论链不可靠，需要补充数据或修正解释"
  rule_2: "step_4 只有一种解释 → 触发双环学习"
  rule_3: "step_6 无法验证 → 整个推论链建立在假设之上，标记为'未经验证的判断'"

双环学习:
  红环: 回到 step_6，收集更多事实
  蓝环: 回到 step_4，生成替代解释
  交汇: 用新事实检验替代解释 → 更新结论
```

## 示例

```
场景: "张三肯定是故意不配合我的项目"

🪜 推论阶梯分析

| 层级 | 内容 | 状态 |
|------|------|------|
| 6 行动 | 准备找领导投诉张三 | — |
| 5 信念 | 张三故意不配合 | ❌ 纯假设 |
| 4 结论 | 他不回复=他不想配合 | ⚠️ 有跳跃 |
| 3 解释 | A: 故意不配合 / B: 他太忙忘了 / C: 邮件被过滤 | ⚠️ 只考虑了 A |
| 2 数据 | 看到：3天没回邮件 / 忽略：他上周请了病假 | ⚠️ 选择性 |
| 1 事实 | 发了邮件，3天未收到回复 | ✅ 可验证 |

薄弱环节: 3 个（信念❌ + 结论⚠️ + 数据⚠️）
建议: 先确认张三是否看到邮件（补充事实），再判断动机
```
