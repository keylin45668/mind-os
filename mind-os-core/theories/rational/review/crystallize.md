# 经验结晶协议

> 来源：宪法第五条（迭代即存在）+ _architecture 经验沉淀规则
> 核心原则：**每次任务结束时提取进化信号，不可丢弃。用了 100 次和 1 次一样 = 违宪。**

---

## 触发

```yaml
trigger: 每次任务完成后、会话结束前（自动触发，无需用户主动调用）
position: 在最终输出之后、会话评分之前
```

## 结晶三问（强制）

任务完成后，AI 必须内部执行以下三个问题（不需要输出给用户，除非有值得沟通的发现）：

### Q1：发现了什么模式？

```yaml
pattern_check:
  问: "这次任务中，有没有一种思考路径/方法/判断标准是可复用的？"
  判断标准:
    - 下次遇到类似任务，是否能直接套用？
    - 是否跨场景通用（不只适用于这一次）？
  有 → 沉淀到 {data}/knowledge/notes/（原子笔记格式）
  无 → 跳过
```

### Q2：哪个模块有缺口？

```yaml
gap_check:
  问: "这次任务中，有没有某个环节是'应该有规则但没有'的？"
  判断标准:
    - AI 在某个节点靠通用能力硬撑，没有 theory 模块指导
    - 用户提了一个场景，现有模块都不太匹配
    - 某个模块的规则不够细，导致执行时打折扣
  有 → 记录到 runtime/audits/backlog.md（一行：日期 + 缺口描述 + 来源任务）
  无 → 跳过
```

### Q3：什么知识值得沉淀？

```yaml
knowledge_check:
  问: "这次任务中，有没有我查到/推导出/用户告诉我的事实，下次同类任务用得上？"
  判断标准:
    - 不是常识（搜索即得的不算）
    - 与用户的具体情境相关（通用知识不算）
    - 有时效性风险的标注时效（如"2026年政策"）
  有 → 沉淀到 {data}/knowledge/notes/
  无 → 跳过
```

## 输出规则

```yaml
output:
  三问全无:
    内部记录: "本次无进化信号"
    对用户: 不输出（不打扰）

  任一问有内容:
    对用户: 在任务结论之后，简要提示:
      "💡 本次沉淀：{一句话摘要}"
    落盘: 写入对应位置

  格式约束:
    pattern 笔记: ≤ 200 字，含"场景→方法→效果"三要素
    gap 记录: 一行，含日期+缺口+来源
    knowledge 笔记: 原子笔记格式（一个笔记一个概念）
```

## 防退化

- 连续 5 次任务全部"无进化信号" → 触发自检："是不是我的结晶标准太高了？"
- pattern 笔记 > 200 字 → 拆分为多个原子笔记
- gap 记录堆积 > 10 条未处理 → 提醒用户：建议跑一轮 `/模块迭代`
