# 结构化学习方法

> 来源：费曼技巧 / 刻意练习 / 能力圈理论 / T 型知识结构
> 核心原则：**学会的标准 = 能教给别人 + 能在新场景中应用**

---

## 触发

```yaml
trigger: 学习/入门/提升/技能/怎么学/想学/学不会/学习路径
```

## 协议

### Step 1：定位（你在哪，要去哪）

```yaml
positioning:
  1. 当前水平: 完全不懂 / 知道概念 / 能操作 / 能教人 / 能创新
  2. 目标水平: 想到哪个层级？（大多数场景"能操作"就够了）
  3. 能力圈映射: 这个领域与你已有能力圈的交集是什么？
     → 交集 = 学习锚点（从已知连接到未知）
  4. 时间预算: 每周能投入多少时间？
```

### Step 2：结构化路径

```yaml
path_design:
  1. 拆解知识结构:
     - 核心概念（必须掌握，其他都建立在此之上）
     - 常用技能（80% 场景用到的 20% 知识）
     - 进阶内容（边用边学，不急）
  2. 排序: 核心概念 → 常用技能 → 进阶（严格串行，不跳级）
  3. 每个阶段设"检查点": 完成什么 = 通过
  4. 推荐资源: 1 本书 / 1 门课 / 1 个练手项目（不要一次推 10 个）
```

### Step 3：费曼学习法（每个知识点）

```yaml
feynman:
  1. 学: 读/听/看（输入）
  2. 教: 用自己的话解释给"完全不懂的人"
     → 解释不清的地方 = 你没真学会 → 回到步骤 1
  3. 简化: 去掉术语，用类比和例子重新表达
  4. 关联: 这个知识点和你已知的什么相似？（类比迁移）
```

### Step 4：刻意练习设计

```yaml
deliberate_practice:
  1. 找到弱点: 哪个环节卡住了？（不要练已经会的）
  2. 设计针对性练习: 专门攻弱点，重复到自动化
  3. 获取反馈: 找人/找系统/找现实结果验证
  4. 调整: 反馈→修正→再练
  周期: 学 → 练 → 反馈 → 调整 → 学（螺旋上升）
```

## 输出格式

```
🎯 学习目标: {技能} — 从 {当前水平} 到 {目标水平}
⏰ 时间: 每周 {N} 小时，预计 {M} 周
🔗 锚点: 与已有能力 {X} 相关联

路径:
  Phase 1（核心概念）: {内容} — 检查点: {能做到什么}
  Phase 2（常用技能）: {内容} — 检查点: {能做到什么}
  Phase 3（进阶）: {按需}

📖 推荐起点: {1 个资源}
⚡ 今天第一步: {具体行动}
```

## 防退化

- 推荐 > 3 个资源 → 选择过载，砍到 1 个
- 没有检查点 → 无法验证学会了没，补上
- 路径没有从已知连接到未知 → 缺锚点，重新设计
