# 输出文档模板（MUST_RUN）

> 迭代引擎最终输出遵循本模板。**总纲言简意赅，详情分章节，单文件 ≤ 1500 tokens。**
> 对话模式下不创建文件，但输出按同样结构组织。

---

## 文件结构

```
runtime/thinking/{task}/output/
  ├── summary.md           ← 总纲（≤ 800 tokens）
  ├── direction-d{N}.md    ← 方向详情
  ├── adversarial-d{N}.md  ← 红蓝对抗记录
  ├── comparison.md        ← 五维对比
  └── appendix.md          ← 附录（可选）
```

---

## Layer 0: 决策卡片（≤ 10 秒速览）

> 决策卡片是所有输出的**最顶层**，位于总纲之上。目标：让决策者扫一眼就能做决定。
> 对话模式下直接输出卡片，文件模式下写入 `summary.md` 顶部。

### 卡片结构（6 个必选字段）

```markdown
## 决策卡片

**结论**：{≤ 30 字，大白话回答"应该怎么做"}

| 看了什么 | {数据来源 / 分析依据} |
|---------|---------------------|
| 发现了什么 | {核心发现} |
| 建议怎么做 | {具体行动} |
| 不做会怎样 | {不作为的后果} |

置信度：{1-5} | 影响面：{低/中/高} | 紧急度：{低/中/高}
```

### 语言规范

- **禁止**使用：赋能、抓手、闭环、矩阵、维度、链路、触达、颗粒度
- **必须**使用：做什么→得到什么 句式
- **数字优先**：说"从60分到85分"而不是"显著提升"
- **像跟同事聊天**：假设读者不懂任何专业术语

### 强制规则

```yaml
decision_card:
  conclusion: 存在 + ≤ 30 字 + 大白话
  fields: 6 个字段全部填写
  language: 禁止抽象词汇 + 数字优先
  position: 总纲之前 / summary.md 顶部
```

### 分层阅读模型

```
Layer 0: 决策卡片（10 秒，够做决定）
Layer 1: 总纲（2 分钟，核心推理）
Layer 2: 方向详情 + 对抗记录（10 分钟+，深挖用）
```

## 总纲（summary.md ≤ 800 tokens）

```markdown
# {任务名} — 分析总纲

> 日期: {date} | 方向: {N}个 | 轮次: {M}轮 | 信心度: {X}%
> 收敛原因: {red_exhausted / all_resolved / ...}

## 一句话结论
{≤ 50 字回答核心问题}

## 推荐方案
{方向} — {理由}
- 存活率: {X}% | 信心度: {Y}/5
- 优势: {1-2 点} | 风险: {1-2 点}

## 各方向速览
| 方向 | 结论 | 存活率 | 状态 |
|------|------|--------|------|

## 下一步行动
1. {行动项 + 截止时间}

## 详细章节
- [方向详情](direction-d1.md) | [对比表](comparison.md) | [对抗记录](adversarial-d1.md)
```

## 方向详情（direction-d{N}.md ≤ 1500 tokens）

```markdown
# 方向 {id}: {名称}

> 状态: {converged/excluded} | 轮次: {N} | 信心度: {X}%

## 本源
{事物本质}

## 最终论点（经红方验证 + 确定性标注）
- [SURVIVED:B-xxx] [事实/推断/待确认] {论点} ✅
- [MODIFIED:B-xxx] [事实/推断/待确认] {论点}（修正: {原因}）⚠️

## 被否决论点
- [FALLEN:B-xxx] {论点}（原因: {红方攻破理由}）❌

## 关键风险（确定性标注）
1. [事实/推断/待确认] {风险} — 来源: [R-xxx]

## 成功条件
1. {条件}
```

## 强制规则

```yaml
validation:
  decision_card: 存在 + 6 字段完整 + ≤ 30 字结论 + 无禁用词
  summary: 存在 + ≤ 800 tokens + 含 5 个必选章节
  direction: 每个 active 方向有对应文件 + 含 [SURVIVED:] 引用
  comparison: 有五维评分表
  all_chapters: ≤ 1500 tokens
  conversation_mode: 先决策卡片 → 再总纲 → 再各方向详情 → 用 --- 分隔
```
