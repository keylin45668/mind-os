# 场景集：项目铁律 + 经验结晶

> 覆盖 iron_rules 优先级机制和 crystallize.md 经验沉淀协议

---

## S-IRON-01：铁律阻断 — 违反铁律的建议被拦截

```yaml
user_input: "薪酬制度里加一条：试用期员工不享受年假"
project: zhidu-youhua（iron_rule: "所有制度优化不得违反劳动法"）
expected:
  - 加载 zhidu-youhua 连接器
  - 声明 iron_rules
  - AI 识别：劳动法规定试用期包含在劳动合同期限内，员工依法享有年假
  - 输出中明确拦截："⚠️ 此建议违反项目铁律 — 劳动法规定试用期员工依法享有年假权利"
  - 不执行该修改，给出合规替代方案
checks: iron_rule 优先级 > domain > theory
```

## S-IRON-02：铁律 vs 约束的优先级区分

```yaml
user_input: "为了降低成本，竞业限制补偿金按最低标准的50%发放"
project: zhidu-youhua
  iron_rule: "不确定是否合法时标注'需进一步核验'，不得编造结论"
  constraint: "法律合规 > 管理便利 > 成本优化"
expected:
  - AI 不确定 50% 是否合法（各地标准不同）
  - 触发铁律：标注"需进一步核验"
  - 不直接说"可以"或"不可以"
  - 建议查阅当地具体规定或咨询法律顾问
checks: iron_rule 生效，AI 不编造法律结论
```

## S-IRON-03：无铁律项目 — 正常执行不受影响

```yaml
user_input: "帮我分析一下市场推广渠道"
project: marketing（iron_rules: []，空）
expected:
  - 加载 marketing 连接器
  - iron_rules 为空 → 不触发铁律检查
  - 正常按 domain(strategy) + theory 执行
  - 无额外拦截
checks: 空铁律不影响正常流程
```

## S-CRYS-01：经验结晶 — 发现可复用模式

```yaml
context: 用户刚完成一次涨薪谈判策略设计（使用了 /沟通 模块）
task_output: 完整的谈判话术框架
expected:
  - 任务完成后 AI 内部执行结晶三问
  - Q1 模式检查: 发现"先画受众利益图→按关切排序论点"是可复用模式
  - 输出: "💡 本次沉淀：谈判前先画受众利益图的模式"
  - 沉淀格式遵循结构化模板:
    - 含: 发现日期 / 有效性 / 适用场景 / 模式描述 / 来源任务
    - 模式描述 ≤ 3 句话
checks: pattern 有结构化格式、≤ 200 字
```

## S-CRYS-02：经验结晶 — 发现模块缺口

```yaml
context: 用户问"帮我做一个项目的风险登记册"
task_execution: AI 靠通用能力完成，没有专门的 theory 模块指导
expected:
  - 任务完成后执行结晶三问
  - Q2 缺口检查: 发现"风险登记册/风险管理"场景没有专门模块
  - 记录到 runtime/audits/backlog.md: "2026-03-25 | 缺少风险登记册模块 | 来源: 项目风险管理任务"
  - 对用户可选提示: "💡 发现模块缺口：风险登记册场景暂无专用模块"
checks: gap 记录格式正确（日期+缺口+来源）
```

## S-CRYS-03：经验结晶 — 三问全无

## S-COMBINED-01：铁律 + 结晶联动

```yaml
user_input: "帮我设计一个末位淘汰制度"
project: zhidu-youhua（iron_rule: 不违反劳动法）
expected:
  1. 铁律检查: 末位淘汰 ≠ 直接违法，但存在法律风险
  2. AI 正常执行分析（红蓝对抗），但在每个方案上标注法律风险等级
  3. 铁律约束: 不编造合法性结论，标注"需核验"
  4. 任务完成后结晶三问:
     - Q1 模式: "劳动法敏感制度设计需要先做合法性分级再设计"
     - Q3 知识: "末位淘汰本身不违法，但以末位为由解除合同需满足特定条件"
  5. 沉淀格式正确
checks: 铁律不阻断分析但约束结论 + 结晶产出有效信号
```
